Motores del proyecto Forseti
Parte lógica o software
Desde cualquier perspectiva de implantación, Forseti, para poder ser accesado, requiere la parte del "Cliente" del modelo "Cliente-Servidor". Este "Cliente" se refiere a un navegador de internet. Es importante recalcar que desde el lado del Cliente, forseti funciona con cualquier sistema sea Windows, Mac, Linux, Solaris etc., lo importante, es que el navegador web soporte frames, java-script, ventanas emergentes y certificados digitales. Actualmente la mayoría no tiene problema alguno y entre los mas comunes contamos con Internet Explorer, Mozilla Firefox, Opera, Safari, Google Chrome, entre otros.

Para la solución En Sitio y En la Nube, el servidor lo vamos a dividir en dos partes:
  1. La base de datos, que es donde se guardan todos los datos de las empresas.
  2. Y la puerta de enlace, que permite el control del flujo de datos entre el navegador web y la base de datos.
Para la base de datos, necesitaremos tener Linux instalado (Cualquier sistema basado en RPM o Debian con nucleo 2,26+) y con el motor de base de datos PostgreSQL en su version 9.3 o posterior.
Para la Puerta de enlace, necesitamos también Linux instalado (Cualquier sistema basado en RPM o Debian con nucleo 2.26+) y los complementos del servidor de los cuales depende forseti para su óptima operación: OpenSSL, OpenJDK 7, RSYNC, Tomcat 7 y el Módulo Forseti. Todos estos complementos se distribuyen bajo las cláusulas de las licencias de la Free Software Foundation.

Aunque parezca una hazaña poner en marcha Forseti, no lo es, ya que teniendo el harware instalado, o un contrato de Amazon Web Services EC2, todo el software se puede poner en marcha en cuestión de minutos. Una vez estando en marcha, Forseti es un sistema extremadamente poderoso y seguro porque se han aislado los datos del controlador y de los usuarios. A este modelo Cliente-Servidor se le denomina "Modelo de Tres Hilos" (1.- Servidor de base de datos, 2.- Servidor de puerta de enlace, 3.- Terminales o Clientes).
