Convenciones
Este manual utiliza las siguientes convenciones tipográficas para marcar ciertas partes del texto:

  • Frases en otro idioma, y textos más importantes se destacan en negrita.
  • Todo lo que representa entrada o salida como, comandos en shell o cmd, código del programa, código SQL y salida en la pantalla, se muestra en azul como esto.

Las siguientes convenciones se utilizan en la sinopsis de un comando, elemento o archivo de la documentación:

  • Entre corchetes "[elemento / comando]" indica partes opcionales.
  • Los puntos "...." significan que un elemento anterior puede repetirse, o que una salida de consola puede dar mas información.
  • Para ser mas específicos, los comandos SQL son precedidos por el símbolo del sistema ">"
  • Los comandos de la terminal o cmd son precedidos por el símbolo "$" que indica el prompt de usuario.
La documentación para la instalación y configuración está enfocada a gente que no conoce Linux. Es por esta razón que está soportada únicamente para la distribución Ubuntu ya que es la que creemos el mejor punto de inicio hacia el salto a Linux. NOTA: Usuarios con cierta experiencia en Linux, podrán instalar en cualquier otra distribución con solo adecuar los comandos a esta. por ejemplo: se usa el comando dpkg para verificar los paquetes existentes lo cual se podrá utilizar otro comando para el mismo fin que soporte tu distribución de Linux.
